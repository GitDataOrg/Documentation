Структура файла content.json

Каждый сайт ZeroNet содержит файл content.json. (Пример файла content.json)

Этот файл кроме всего прочего содержит список всех файлов вашего сайта и подпись, созданную с помощью вашего секретного ключа. Это необходимо во избежание подделки файла (то есть только вы, или люди, которым вы доверяете, можете обновлять содержимое сайта).

Далее список поддерживаемых функций:

Автоматическая генерация

address

Адрес вашего сайта

Пример: 1TaLk3zM7ZRskJvrh3ZNCDVGXvkJusPKQ

address_index

Ключ BIP32 адреса вашего BIP32-источника. Автоматически добавляется когда вы создаете копию сайта. Это позволяет восстановить секретный ключ сайта от вашего BIP32-источника.

Пример: 30926910

cloned_from

Адрес исходного сайта, который вы скопировали.

Пример: 1BLogC9LN4oPDcruNz3qo1ysa133E9AGg8

files

Размер и значение кэш sha512 автоматически загруженных файлов, содержащих ваш сайт. Автоматически создается при использовании команды zeronet.py siteSign siteaddress privatekey.

Пример:

    "css/all.css": {
      "sha512": "869b09328f07bac538c313c4702baa5276544346418378199fa5cef644c139e8",
      "size": 148208

files_optional

Размер и значение кэш sha512 дополнительных файлов вашего сайта. Автоматически создается при использовании команды zeronet.py siteSign siteaddress privatekey.

Пример:

    "data/myvideo.mp4": {
      "sha512": "538c09328aa52765443464135cef644c144346418378199fa5cef61837819538",
      "size": 832103

modified

Время создания content.json.

Пример: 1425857522.076

sign (deprecated)

Подпись ECDSA файла content.json, содержащий контент (ключи отсортированы без пробелов и записей sign и signers_sign). Для обеспечения обратной совместимости в ближайшее время будут удалены.

Пример:

  "sign": [
    43117356513690007125104018825100786623580298637039067305407092800990252156956,
    94139380599940414070721501960181245022427741524702752954181461080408625270000
  ],

signers_sign

Адреса доступных подписантов корневого файла content.json, подписанного с использованием секретного ключа адреса сайта (возможна множественная подпись)

Формат подписанной строки: [signers_required]:[signer address],[signer address]

Пример: HKNDz9IUHcBc/l2Jm2Bl70XQDL9HYHhJ2hUdg8AMyunACLgxyXBr7EW1/ME4hGkaFZSFmIxlInmxH+BrMVXbnLw=

signs

Подписи ECDSA файла контента content.json (ключи отсортированы без пробелов и записей sign и signers_sign).

Пример:

  "signs": {
    "1TaLk3zM7ZRskJvrh3ZNCDVGXvkJusPKQ": "G6/QXFKvACPQ7LhoZG4fgqmeOSK99vGM2arVWkm9pV/WPCfc2ulv6iuQnuzw4v5z82qWswcRq907VPdBsdb9VRo="
  },

zeronet_version

Версия ZeroNet, использованная для генерации файла content.json.

Пример: 0.2.5

Настройки

background-color

Цвет фона обертки

Пример: #F5F5F5

cloneable

Резрешение создании копии сайта, если указано true.

Для того, чтобы ваш сайт правильно клонирован, вы должны добавить файлы данных для чистого запуска (например, без сообщений в блоге). Для этого добавьте суффикс -default к файлам данных и каталогам. В процессе клонирования каждый файл и каталог, имеющий альтернативный файл с суффиксом -default, будет пропущен, а у скопированных суффикс -default удален.

description

Описание вашего сайта, отображаемое нв заголовком в ZeroHello.

Пример: Децентрализованный демо-форму

domain

Доменной имя вашего сайта в системе Namecoin. ZeroHello будет привязан к этому имени в случае установки у пользователя плагина Zeroname.

Пример: Blog.ZeroNetwork.bit

ignore

Игнорирование фалов при подписании, соответсвующих regexp шаблону.

Пример: ((js|css)/(?!all.(js|css))|data/users/.*) (исключит все файлы js и css за исключение all.js и all.css, а также исключит содержимое папки data/users/ directory)

includes

Подключает другой content.json

Пример:

{
  "data/users/content.json": {
    "signers": [ # Адреса возможных подписантов для файлов
      "1LSxsKfC9S9TVXGGNSM3vPHjyW82jgCX5f"
    ],
    "signers_required": 1 # Количество подтвержденных подписей, тербуемых для утверждения файла (возможна множественная подпись),
    "files_allowed": "data.json", # Шаблон regexp утвержадаемых файлов в подключаемом файла
    "includes_allowed": false, # Разрешение вложенных подключений файлов
    "max_size": 10000, # Максимальный допустимый объем вложений (в байтах)
  }
}

merged_type

Источник данных для указанного типа объединения сайта

Пример: ZeroMe

optional

Шаблон regexp для выборки прилоежнных файлов

Пример: (data/mp4/.*|updater/.*) (файлы из директории data/mp4 и updater)

signs_required

Количество подтвержденных подписей, тербуемых для утверждения файла (возможна множественная подпись)

Пример: 1

title

Заголовок сайта, отображаемый в заголовке браузера и на странице ZeroHello.

Пример: ZeroTalk

user_contents

Правила разрешенного пользовательского контента текущего каталога.

Параметр Описание
cert_signers Разрешенные домены и адреса возможных подписантов
permission_rules Разрешенные имена файлов и общий размер папок, использующих в качестве базового сертифицированный домен или метод авторизации
permissions Права доступа пользователей (false = пользователь заблокирован).

Пример:

  "user_contents": {
    "cert_signers": {
      "zeroid.bit": [ "1iD5ZQJMNXu43w1qLB8sfdHVKppVMduGz" ]
    },
    "permission_rules": {
      ".*": {
        "files_allowed": "data.json",
        "max_size": 10000
      },
      "bitid/.*@zeroid.bit": { "max_size": 40000 },
      "bitmsg/.*@zeroid.bit": { "max_size": 15000 }
    },
    "permissions": {
      "bad@zeroid.bit": false,
      "nofish@zeroid.bit": { "max_size": 100000 }
    }
  }

viewport

Содержание мета-тега viewport (используется для мобильных устройств).

Пример: width=device-width, initial-scale=1.0
